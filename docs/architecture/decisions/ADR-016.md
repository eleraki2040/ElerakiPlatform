# ADR-016

Title:
xUnit and FluentAssertions

Status:
Accepted

Date:
2026-07-05

Owner:
Mohamed Eleraki (Project Owner)

---

## CONTEXT

The platform needs a testing framework and assertion library that promote readable, maintainable tests.

## DECISION

We will use **xUnit** as the testing framework and **FluentAssertions** as the assertion library.

## RATIONALE

1. **Modern Framework:** xUnit is the preferred framework for modern .NET projects
2. **Readability:** FluentAssertions provides natural language-style assertions
3. **Extensibility:** Both frameworks support custom extensions and assertions
4. **Community:** Strong adoption and excellent documentation
5. **Integration:** Works seamlessly with CI/CD pipelines and code coverage tools

## CONSEQUENCES

### Positive
- Tests are more readable and self-documenting
- Reduced boilerplate in assertion code
- Excellent support for parallel test execution
- Strong ecosystem and community support

### Negative
- FluentAssertions adds another dependency
- Requires learning the fluent API syntax
- Error messages can be verbose for complex assertions
- Team must be consistent with assertion style
