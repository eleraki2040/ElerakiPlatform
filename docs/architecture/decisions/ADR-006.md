# ADR-006

Title:
CQRS with MediatR

Status:
Accepted

Date:
2026-07-05

Owner:
Mohamed Eleraki (Project Owner)

---

## CONTEXT

The platform needs a consistent way to separate read and write operations while maintaining clean, testable command and query handlers.

## DECISION

We will implement the **CQRS** pattern using **MediatR** for dispatching commands and queries.

## RATIONALE

1. **Separation of Concerns:** Read and write operations are handled independently
2. **Testability:** Commands and queries can be tested in isolation
3. **Maintainability:** Business logic is centralized in handlers
4. **Flexibility:** Easy to optimize queries separately from commands
5. **Ecosystem:** MediatR is widely adopted with excellent .NET integration

## CONSEQUENCES

### Positive
- Clean separation between reads and writes
- Reduced complexity in controllers and services
- Improved testability of business logic
- Consistent pattern across all features

### Negative
- Additional abstraction layer to learn
- More files for simple operations
- Requires discipline to keep queries and commands synchronized
