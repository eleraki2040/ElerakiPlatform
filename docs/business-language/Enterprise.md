# Enterprise

المركز/الكيان الأساسي الذي تنتمي إليه كل بيانات المنصة.

## التعريف

Enterprise ليس مجرد "Company" أو "Organization".
هو يمثل أي كيان أعمال مستقل يمكنه استخدام المنصة.

## أمثلة

- شركة خاصة
- مستشفى
- مدرسة/جامعة
- مصنع
- وزارة
- جمعية
- مؤسسة خيرية

## القواعد

- Enterprise هو الجذر الحقيقي (Aggregate Root) للمنصة.
- كل بيانات المنصة مرتبطة بـ Enterprise واحد على الأقل.
- No Engine يعدل بيانات Enterprise آخر مباشرة.
- أي تواصل بين Engines يحمل معه EnterpriseIdentifier.

## الحقول الأساسية

| الحقل | النوع | الوصف |
|------|------|--------|
| Id | Guid | معرف فريد |
| Code | string | معرف أعمال فريد مثل `ELK`, `HQ`, `ORG-001` |
| Name | string | اسم المؤسسة |
| LegalName | string | الاسم القانوني (اختياري، يمكن لاحقاً كـ Profile) |
| Status | EnterpriseStatus | حالة المؤسسة |

## الحالات (Status)

- Active
- Inactive
- Suspended
- Archived

## القاعدة الذهبية

كل خاصية نضيفها إلى Enterprise يجب أن نجيب عن سؤال:
> هل تحتاجها جميع أنواع المؤسسات؟

إذا كانت الإجابة "لا"، فلا نضعها في النواة.
