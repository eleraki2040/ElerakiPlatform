# ADR-019

Title:
Swagger/OpenAPI Documentation

Status:
Accepted

Date:
2026-07-05

Owner:
Mohamed Eleraki (Project Owner)

---

## CONTEXT

API consumers need a way to discover, understand, and test the platform's HTTP endpoints without reading source code.

## DECISION

We will use **Swagger (OpenAPI)** to generate interactive API documentation for all HTTP endpoints.

## RATIONALE

1. **Discoverability:** Developers can explore available endpoints and schemas
2. **Interactivity:** Built-in testing allows consumers to try endpoints directly
3. **Automation:** Documentation is generated from code and stays in sync
4. **Standardization:** OpenAPI is an industry standard for REST APIs
5. **Integration:** Swashbuckle provides seamless ASP.NET Core integration

## CONSEQUENCES

### Positive
- Reduced need for separate API documentation
- Faster onboarding for new API consumers
- Early detection of API contract issues
- Supports client code generation from the specification

### Negative
- Requires maintenance of XML comments and attributes
- Swagger UI adds another endpoint to secure
- May expose internal implementation details if not configured carefully
- Additional bundle size and startup time
