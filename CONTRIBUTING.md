## PURPOSE

This document provides guidelines for contributing to the Eleraki Platform.

---

## CONTRIBUTION GUIDELINES

### 1. Documentation First
All contributions must be documented before implementation.

### 2. Architecture Review
All architectural changes must be approved through an ADR.

### 3. Business First
Every feature must solve a real business problem.

### 4. Code Quality
- Follow existing patterns
- Write tests
- Keep it simple
- Document decisions

---

## SUBMITTING CHANGES

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Submit a pull request
5. Get approval from the team
