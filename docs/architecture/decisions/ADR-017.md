# ADR-017

Title:
SQLite for Development

Status:
Accepted

Date:
2026-07-05

Owner:
Mohamed Eleraki (Project Owner)

---

## CONTEXT

Local development requires a database that is easy to set up, portable, and requires minimal configuration.

## DECISION

We will use **SQLite** as the database for local development environments.

## RATIONALE

1. **Zero Configuration:** No separate database server required
2. **Portability:** Database is a single file that can be shared or reset
3. **Speed:** Fast startup and operation for development workflows
4. **Compatibility:** Fully supported by Entity Framework Core
5. **Cost:** No licensing or infrastructure costs

## CONSEQUENCES

### Positive
- New developers can start coding immediately
- Database migrations work the same as production
- CI/CD pipelines can use SQLite for fast test runs
- Reduced local resource usage

### Negative
- SQLite behavior differs from SQL Server in edge cases
- Not representative of production performance characteristics
- Limited concurrency compared to server databases
- Must ensure compatibility with production database features
