# ADR-008

Title:
Entity Framework Core as ORM

Status:
Accepted

Date:
2026-07-05

Owner:
Mohamed Eleraki (Project Owner)

---

## CONTEXT

The platform requires a robust, maintainable data access layer that works with multiple database providers and integrates well with .NET.

## DECISION

We will use **Entity Framework Core** as the primary ORM for all data access operations.

## RATIONALE

1. **Productivity:** Rapid development with LINQ-based queries and migrations
2. **Cross-Database:** Supports SQLite, SQL Server, PostgreSQL, and others
3. **Maintainability:** Schema changes tracked via migrations
4. **Ecosystem:** First-class .NET support with excellent tooling
5. **Flexibility:** Can be used with or without the repository pattern

## CONSEQUENCES

### Positive
- Faster development of data access code
- Database migrations simplify schema evolution
- Strong tooling support in Visual Studio and Rider
- Reduced SQL boilerplate for common operations

### Negative
- Potential performance overhead for complex queries
- Requires understanding of EF Core behavior to avoid pitfalls
- Migrations must be managed carefully in team environments
- May generate inefficient SQL for advanced scenarios
