# ADR-018

Title:
SQL Server for Production

Status:
Accepted

Date:
2026-07-05

Owner:
Mohamed Eleraki (Project Owner)

---

## CONTEXT

Production deployments require a robust, scalable, and fully managed relational database with enterprise features.

## DECISION

We will use **SQL Server** as the primary database for production deployments.

## RATIONALE

1. **Reliability:** Proven enterprise-grade database with high availability features
2. **Scalability:** Supports high throughput and large datasets
3. **Integration:** Native support for Entity Framework Core and .NET
4. **Ecosystem:** Extensive tooling, monitoring, and management capabilities
5. **Compliance:** Meets enterprise security and compliance requirements

## CONSEQUENCES

### Positive
- High performance and reliability for production workloads
- Rich feature set including stored procedures and advanced indexing
- Strong support for backup, recovery, and high availability
- Familiar tooling for database administrators

### Negative
- Licensing costs for production deployments
- Requires infrastructure setup and maintenance
- Higher resource requirements compared to SQLite
- Connection management must be carefully configured
